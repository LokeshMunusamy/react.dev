### Overview {/*overview*/}

To provide a robust version control system for test cases in QATouch, enabling users to track changes, maintain historical records, and manage test case versions efficiently.

**Note:** Feature only available for Professionals and Enterprise customers

### Key Features {/*key-features*/}

1. Version Tracking:
Each test case is assigned a unique version identifier. Any changes made to a test case (such as updates to Title, Description, Precondition Tags, Reference, Test data, Type, Priority and Mode) result in the creation of a new version.

2. Change Logs
Detailed logs are maintained for each version, documenting what changes were made, by whom, and when.

3. Comparison and Diff Views:
Users can compare different versions of a test case to see what has changed. A diff view highlights the additions, deletions, and modifications between versions.


### How to Use Test Case Version History ? {/*how-to-use-test-case-version-history-*/}
1. Navigate to the Test Case Page. Create a Test Case.
2. Now click the Edit icon, and an overlay will open.
3. Update the values of the case Title, Description, Precondition, Tags, Reference, Test Data, Type, Priority, and Mode.
4. Then click Save.

5. Click Three Dots More action menu in the test case, in the menu Click Test Case Version History.

![versions2 image](../../assets/faq-screenshots/versions/2.png)

6. The overlay contains Date, Duration, Updated By and Action columns.

![versions4 image](../../assets/faq-screenshots/versions/4.png)

7. Click the View Icon, Now you can see the Updated version of Test Case.

8. The overlay contains Detailed logs maintained for Test Case, documenting what changes were made, by whom, and when.

![versions3 image](../../assets/faq-screenshots/versions/3.png)

### Step Level - Version History {/*step-level---version-history*/}

The Step Level Version History feature allows you to track and manage changes made to individual steps within your test cases. This ensures better, helping you maintain a clear audit trail . Access the complete log of changes, including details such as who made the changes, when they were made, and what was updated.

### How to Access Step Level Version History: {/*how-to-access-step-level-version-history*/}
1. Open the test case you wish to review.
2. Navigate to the specific test step.
3. Click on edit and change the test case steps and save
4. Then , Click on the Version History option
5. View the complete log of changes, including details such as who made the changes, when they were made, and what was updated.
6. In the Accordion section of each values separated by Previous data, Current data and Difference, it’s helpful for easily identify the changes
