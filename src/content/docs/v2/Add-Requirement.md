### To add Requirements for a Project in QA Touch, follow these steps: {/*to-add-requirements-for-a-project-in-qa-touch-follow-these-steps*/}

1. Login to QA Touch
2. On the left panel, Click on Project List
3. Click on the project you want to add requirements
4. From the project submenu , click on the Requirements option
5. Click on Add Requirement Type.
6. Fill in the Requirement type and click Save.
7. Click on the Add button in Requirement Type Menu to add a requirement(s).
8. Fill in the Requirement Such as: -Title -Description -Files (if any) -then click Save.
9. The requirement has now been created under the selected project.

### Import Requirements in Bulk {/*import-requirements-in-bulk*/}

In the Requirements page,

1. Click on the Import button in the right
2. Download sample Requirements in .csv format
3. In the .csv file, fill in the -Requirement Type -Requirement Title -Description -save it.
4. Upload the file in QA Touch.
5. Now You can also import the Requirement Types with hierarchies from the import file. Simply use greater-than symbol (>) in the first column as a separator.

Example : Requirement Type > Sub-Requirement Type 1 > Sub-Requirement Type 2

All the requirements in the .csv file will appear in your QA Touch Requirements page.

### Export Requirements {/*export-requirements*/}

To export Project Requirements,

1. Click on the Export button in the right end of the requirements page
2. You can export in any one of the -.xls -.xlsx -.csv formats

![export-requirement file](../../assets/faq-screenshots/add-requirement/export-requirement.gif)

You can either all requirements or a specific requirement type.

Click on the right of Traceability matrix and select either -Export -Export All

If you select Export, it will only export the selected Requirement Type.
If you select Export All, it will export all the Requirements in the project.

The exported Requirements file will contain:

1. Requirement Id
2. Document Title (Requirement Type)
3. Title (Requirement title)
4. Description (Requirement Description)
5. No of linked Test Cases Created Date
6. Updated Date

### Requirement Approve/Reject/Re-verify {/*requirement-approverejectre-verify*/}
The Requirement Page allows users to manage requirements efficiently by providing options to Approve, Reject, or Re-Verify them along with comments for better tracking and communication.

![add-requirement1 image](../../assets/faq-screenshots/add-requirement/1.png)

1. **Approve** – Mark the requirement as approved for further processing.
2. **Reject** – Decline the requirement with a reason, ensuring clarity.
3. **Re-Verify** – Send the requirement for re-evaluation before final approval.


![add-requirement2 image](../../assets/faq-screenshots/add-requirement/2.png)
![add-requirement3 image](../../assets/faq-screenshots/add-requirement/3.png)

### How to Approve,Reject and Re-verify a Requirement? {/*how-to-approvereject-and-re-verify-a-requirement*/}

1. Goto Requirement page, click approval pending option
2. Approve or reject the requirement by selecting the required option
3. To Notify the user ,tag the assigned use and comments
4. Click on save ,the status will be updated successfully.

![add-requirement6 image](../../assets/faq-screenshots/add-requirement/6.png)

### How to view the status history of requirement approval? {/*how-to-view-the-status-history-of-requirement-approval*/}

To view the status of the requirement Approved, Rejected and Re-verify click on the approval history icon.All the status and comments will be displayed.

![add-requirement4 image](../../assets/faq-screenshots/add-requirement/4.png)
![add-requirement5 image](../../assets/faq-screenshots/add-requirement/5.png)

### How to bulk update the approval status? {/*how-to-bulk-update-the-approval-status*/}

To Bulk update the approval status, select the required requirements and select the status (Approved,Rejected and Re-verify ) and click on save .The selected requirements will be updated successfully.
