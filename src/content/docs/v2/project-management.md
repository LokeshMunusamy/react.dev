#### How to Create, Edit, and Delete a Project in QA Touch? {/*project-section-2*/}

1. Login to your QA Touch account.
2. After logging in, you’ll be directed to the Projects page.
3. Select the “Add Project” button.
4. Fill out essential details like:
5. Choosing your workspace
6. Entering project name
7. Finally, click on save.
8. Click Submit to save your project.

Once created, the project becomes available in your main project list and is ready to house your test cases and sprints.

#### Understanding Project status? {/*understanding-project-status*/}

Each project in QA Touch can be assigned a status, allowing better lifecycle tracking and team filtering:

Ongoing – Currently being worked on. Completed – Fully executed, with no further changes expected. Archived – Stored out of view for reference, not active.

Use statuses to organize your dashboard, prioritize work, and filter reports.


#### How to Assign user to a Project in QA Touch? {/*how-to-assign-user-to-a-project-in-qa-touch*/}
The Assign User functionality streamlines collaboration by allowing you to assign users to specific projects.

1. On the project details page, locate and click on the "Assign User" option.
2. A popup window will appear with two tabs: "Add Assign User" and "Assigned User."
3. In the "Add Assign User" tab:
4. Select the appropriate role for the user from the available options.
5. Choose the user you want to assign to the project.
6. Click the "Save" button to confirm and save the user assignment.
7. To ensure the user is assigned successfully, navigate to the "Assigned Users" tab.
When you create a user as a tester/developer in the system, you have to assign them to the project to give access.

#### Editing an Existing Project {/*editing-an-existing-project*/}

1. From the Projects dashboard, locate the project.
2. Click on the Edit icon.
3. You’ll be redirected to the project settings page.
4. Update any of the following: Project Title Description User Assignments Status (Ongoing, Completed, Archived)
5. Visual Customizations (Background color, font color, logo)
6.After making the necessary changes, scroll down and click Update.


#### What is a pin in QA Touch? {/*what-is-a-pin-in-qa-touch*/}

The Pin feature allows you to prioritize and easily access your most important projects. By pinning a project, you ensure it stays at the top of your project list, making it readily identifiable and accessible.

Look for the pin icon next to Assign user.

Select the project you want to prioritize and click on the pin icon next to Assign user. Once clicked, the icon changes its appearance to indicate that the project is now pinned.

Your pinned project will now be prominently displayed at the top of the project list.


#### How to unpin a Project? {/*how-to-unpin-a-project*/}

If you need to remove a project from the pinned list, simply click on the pin icon again located next to the project name. The project will be unpinned and will move back to its original position in the project list.


#### What is Ongoing/Completed/Archived Projects? {/*what-is-ongoingcompletedarchived-projects*/}

Efficiently organize and track the status of your projects with our status management features. Whether your project is ongoing, completed, or archived, you can easily categorize and locate it as needed. Here's a step-by-step guide on how to use these features: On the project list page, you'll find a dropdown menu placed before the "Add User" option. Choose from three status options:

Ongoing: Represents projects that are currently in progress.

Completed: Marks projects that have been successfully finished.

Archived: Indicates projects that you want to hide or have deleted.


#### How to Archiving a Project? {/*how-to-archiving-a-project*/}

1. To archive a project, click on the "Delete" option placed next to "Edit" in the project details.
2. A popup window will appear with three options: Archive, Delete, and Cancel.
3. Select "Archive," and the project will be automatically moved to the archived status.
4. If you want to view or manage archived projects, go to the dropdown and select "Archived Projects." Your archived projects will be visible.


#### How to change Project Status: {/*how-to-change-project-status*/}

1. To change the status of a project, click on the "Edit" option placed before "Delete" in the project list page.
2. Redirected to the project edit page.
3. Locate the "Status" option and choose from the available options: Ongoing, Completed, or Archived.
4. Scroll down and click on the "Update" button to save the changes.
5. The project will now be moved under selected status. By effectively utilizing these status management features, you can enhance your project management experience and keep your workspace organized.


#### How to Delete project? {/*how-to-delete-project*/}

Need to remove an outdated or completed project? QA Touch gives you the option to delete projects permanently.

Open the Projects list.

Locate the project you wish to remove.

1. Click the Delete icon next to the project name
2. A pop-up window will appear with three options: Archive, Delete, and Cancel.
3. Click Delete to permanently remove the project from your workspace.
4. Confirm the deletion when prompted.

Deleted projects are permanently removed and cannot be recovered. Archive instead if you think the project might be needed later.

##### Important Note: {/*important-note*/}

Deleting a project is an irreversible action, and all associated data will be permanently removed. Please make sure you have considered the implications before confirming the deletion.
