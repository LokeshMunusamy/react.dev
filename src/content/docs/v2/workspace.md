### Workspace/ Group / Team {/*workspace-group--team*/}

This feature allows users to organize their projects and collaborate effectively within designated workspaces, groups, or teams.

Note: Workspace feature supported Professional / Enterprise plan.

![workflow image](../../assets/faq-screenshots/workspace/workflow.png)

### What is Workspace/Group/Team {/*what-is-workspacegroupteam*/}

Workspace management is a comprehensive platform that allows organizations to efficiently manage their office spaces and projects. Workspaces serve as central hubs for teams to manage their work, fostering collaboration and enhancing workflow efficiency across projects and teams. Additionally, workspace management is easily accessible to authorized users, enhancing workflow efficiency and organizing projects efficiently.

### How to create a Workspace/Group/Team? {/*how-to-create-a-workspacegroupteam*/}

A Super Administrator and an admin who has complete access to all control to create, edit, and delete the workspace. Workspace management is supported only for professional plans. To create a workspace follow the below steps:

1. Login to QA Touch.
2. On the left panel, click on Administration -> Workspace menu
3. Click on Add workspace/Group/Team button
4. Enter the workspace name
5. Click on Save, and the workspace will be created successfully.

![workspace image](../../assets/faq-screenshots/workspace/workspace.png)

### How to Edit a Workspace/Group/Team? {/*how-to-edit-a-workspacegroupteam*/}

1. To Edit a workspace, click on the pencil icon on the workspace list page.
2. An edit pop-up will appear, Enter the workspace name and click save
3. Changes will be updated successfully

### How to Archiving a Workspace? {/*how-to-archiving-a-workspace*/}

1. Click on the "Archive" option near by Permission icon.
2. A popup window will appear with Archive Option, it will soft delete you can revamp back any time.
3. Click on the "Archive" option near by Add workspace button
4. List all the archive list of records. Please click the select All option Revert All button.

### How to assign a workspace in a project? {/*how-to-assign-a-workspace-in-a-project*/}

1. To assign a workspace, Login to QA Touch as Super admin/admin
2. Click on the Add Projects button on the project list page
3. On the add project pop-up, select a workspace in the dropdown
4. Then enter the project name, click on save
5. The project will be moved under the selected workspace.

![workspace-assign image](../../assets/faq-screenshots/workspace/workspace-assign.png)

![workspace-edit-assign image](../../assets/faq-screenshots/workspace/workspace-edit-assign.png)

### How to Filter the workspace? {/*how-to-filter-the-workspace*/}

1. To filter the workspace, on the project list page click the dropdown.
2. It will list all the workspaces created
3. Select a workspace, projects under the selected workspace will be displayed.


![workspace-list image](../../assets/faq-screenshots/workspace/workspace-list.png)

### Workspace Permission {/*workspace-permission*/}

QA Touch offers the option to customize the access for the workspace. By default, All admins will have access to View / Create projects within the workspace.

1. The Grant/ Remove access,
2. Go to Administration->Workspace
3. Click on Permissions next to the respective workspace
4. Here you can Turn On/Off the access permissions
5. Note: Admins without access to the workspace will not be able to create/edit projects within that workspace.

![permission image](../../assets/faq-screenshots/workspace/permission.png)
